What is NoSQL?

NoSQL( Not Only SQL) is a non relational database that is used to store large or vast amounts of data. It is best used for storing non relational data and scaling out rather than up and as a result saves large companies millions every year. There are no schemas involved, no long advanced queries, and it makes data storage extremely scalable and highly available.

Relational Vs. Non-relational|

Relational databases are databases that operate on a relational model which have tables that contain rows and columns. Each table, which is sometimes called a relation, in a relational database contains one or more data categories in columns, also called attributes. Each row, also called a record or tuple, contains a unique instance of data, or key, for the categories defined by the columns. Each table has a unique primary key, which identifies the information in a table. The relationship between tables can then be set via the use of foreign keys -- a field in a table that links to the primary key of another table.

Vertical scaling. Relational databases scale vertically which means that you scale by adding more power (CPU, RAM) to an existing machine and therefore build up. This is very expensive and requires large facilities to hold the infrastructure. Vertical-scaling is often limited to the capacity of a single machine, scaling beyond that capacity often involves downtime and comes with an upper limit.

A non-relational database is any database that does not follow the relational model provided by traditional relational database management systems. This category of databases, also referred to as NoSQL databases, has seen steady adoption growth in recent years with the rise of Big Data applications. NoSQL databses use a key-value store and do not require complex queries like join operations in order to retrieve information. They are extremely flexible and can retrieve information at an extremely fast rate.

Horizontal scaling means that you scale by adding more machines into your pool of resources. An easy way to remember this is to think of a machine on a server rack, we add more machines across the horizontal direction and add more resources to a machine in the vertical direction. In a database world horizontal-scaling is often based on the partitioning of the data i.e. each node contains only part of the data, in vertical-scaling the data resides on a single node and scaling is done through multi-core i.e. spreading the load between the CPU and RAM resources of that machine. With horizontal-scaling it is often easier to scale dynamically by adding more machines into the existing pool - Vertical-scaling is often limited to the capacity of a single machine, scaling beyond that capacity often involves downtime and comes with an upper limit.

NoSQL Today|

Today NoSQL is all about the cloud. From NoSQL databases like BigTable, DynamoDB, and CosmosDB to MongoDB's Atlas, cloud is the way to go for big data. Why don't companies just host the NoSQL DB on-premise? well...

Think about scaling? Even though you can scale with a NoSQL database a lot easier than a relational one it still costs. In the long run, you will still have to upgrade, maintain, and secure your systems. Cloud solves all of these issues and does so in a better way.

With cloud, you can grow as large as you want and keep paying that same cost you got in for via contract, this way, you save more money. NoSQL does so well in cloud because of that and the fact that building fault tolerant infrastructures is easy. Don't believe me? Let's see...
